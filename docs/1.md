# Example GitHub Repository for ez-appsec Integration

This guide demonstrates how to set up ez-appsec security scanning in a GitHub repository.

## Repository Structure

```
my-repo/
├── .github/
│   └── workflows/
│       └── github-scan.yml
├── .ez-appsec.yaml  (optional)
└── README.md
```

## Setup Steps

### 1. Clone and Initialize

```bash
# Create a test repository
git init

# Create a simple file to scan
cat > example.py << 'EOF'
# Example file with a vulnerability
api_key = "EXAMPLE-AWS-ACCESS-KEY-ID"

def get_data():
    return {"key": api_key}
EOF
git add example.py
git commit -m "Add example code with vulnerability"
```

### 2. Install ez-appsec Workflow

```bash
# Install the ez-appsec GitHub skill
curl -fsSL https://raw.githubusercontent.com/jfelten/ez-appsec/main/skills/install.sh | bash -s --github

# This creates:
# - .github/workflows/github-scan.yml
# - Sets EZ_APPSEC_VERSION variable
# - Creates a pull request for review
```

### 3. Verify Workflow File

Your `.github/workflows/github-scan.yml` should look like:

```yaml
name: Security Scan

on:
  push:
    branches:
      - main
  pull_request:

permissions:
  contents: read
  security-events: write

env:
  EZ_APPSEC_VERSION: "latest"

jobs:
  scan:
    runs-on: ubuntu-latest
    container:
      image: jfelten/ez-appsec:\${{ env.EZ_APPSEC_VERSION }}
    steps:
      - name: Checkout
        uses: actions/checkout@v4

      - name: Run ez-appsec scan
        run: |
          mkdir -p scan-results
          ez-appsec github-scan . --output scan-results/ez-appsec.sarif

      - name: Upload SARIF to GitHub Security
        uses: github/codeql-action/upload-sarif@v3
        with:
          sarif_file: scan-results/ez-appsec.sarif
          continue-on-error: true
```

### 4. Test the Integration

```bash
# Push to trigger a scan
git add .
git commit --allow-empty -m "Test ez-appsec scan"
git push

# The workflow will run automatically and you can check:
# 1. Actions tab - see the scan workflow
# 2. Security tab - view SARIF results (requires GitHub Advanced Security)
# 3. Workflow run artifacts - download full scan results
```

## Expected Results

### SARIF Output

The workflow will generate `ez-appsec.sarif` with this structure:

```json
{
  "version": "2.1.0",
  "runs": [{
    "tool": {
      "driver": {
        "name": "ez-appsec",
        "version": "0.1.17",
        "rules": [...]
      }
    },
    "results": [{
      "ruleId": "gitleaks-rule-id",
      "level": "error",
      "message": {"text": "Secret detected"},
      "locations": [...]
    }]
  }]
}
```

### Scan Summary Comment

On pull requests, the workflow will add a comment like:

```
🔒 Security Scan Results

**Findings:** 2

📊 [View full scan results](https://github.com/owner/repo/actions/runs/12345)

💡 Use `/ez-appsec-scan` in your local environment for detailed analysis.
```

### Artifacts

The workflow uploads scan artifacts including:
- `ez-appsec.sarif` - SARIF format for GitHub Security tab
- `vulnerabilities.json` - JSON format for dashboard integration

Artifacts are retained for 7 days.

## Advanced Configuration

### Set Version Pinning

To pin to a specific ez-appsec version instead of `latest`:

```bash
# Set repository variable
gh variable set EZ_APPSEC_VERSION "0.1.17" --repo=owner/repo

# Update workflow to use variable
# In .github/workflows/github-scan.yml:
# env:
#   EZ_APPSEC_VERSION: ${{ vars.EZ_APPSEC_VERSION }}
```

### Dashboard Integration

For multi-project dashboards, configure the dashboard repository:

```bash
# Set the dashboard repository variable
gh variable set EZ_APPSEC_DASHBOARD_REPO "owner/ez-appsec-dashboard" --repo=owner/repo

# The scan workflow will automatically push results to the dashboard
# when EZ_APPSEC_DASHBOARD_REPO is configured
```

## Troubleshooting

### Workflow Not Triggering

1. **Check YAML syntax**: Ensure `.github/workflows/github-scan.yml` has valid YAML (no tabs)
2. **Check permissions**: Repository needs Actions enabled
3. **Check branch**: Default branch in workflow must match your main branch
4. **Verify variables**: Use `gh variable list` to check `EZ_APPSEC_VERSION`

### SARIF Not Appearing in Security Tab

1. **GitHub Advanced Security**: Required for Security tab display
2. **SARIF format**: Verify the SARIF file is valid JSON
3. **File path**: Ensure the `sarif_file` parameter points to the correct path

### Scan Finding Nothing

If the scan finds 0 vulnerabilities:
- Check your codebase has detectable vulnerabilities
- Try adjusting severity filter
- Use a test file that should trigger findings
